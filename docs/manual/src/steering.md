# 足回り

このモジュールは`seri/steering.h`として公開されている。主に足回り制御の関数などを集約している。

現在、対応している足回りは以下の通りである。

- クローラー(2輪)
- メカナムホイール(4輪)
- オムニホイール(n輪)

## 足回りパラメータ

足回りに関するパラメータをまとめた型として`steering_param_t`を実装している。これはタイヤの最高回転速度、速度における並進速度・回転速度の比率、タイヤ半径、ロボット中心からタイヤまでの距離を保持する。
本ライブラリにおいて、並進移動と回転移動を両立させるためのアプローチとして、速度の内それぞれが占める割合を事前に決めておくという手法を取った。この場合、それぞれが最大値となった場合でも最終的な出力値をタイヤの最大値に収めることができる。ただしその代償としてそれぞれ単独で行う場合に最高パフォーマンスを得ることが出来ないため、例えば並進・回転をモード切替するようにし、それぞれの比率が最高になるような`steering_param_t`を用意し切り替えるようにするなどの対策が必要である。

## 足回り速度計算

足回りの速度計算用関数として`calc_speed_*`系関数を定義している。計算結果格納用配列(サイズは輪数)、`pid_data_t`へのポインタの配列(サイズは輪数)、`steering_param_t`、`sticks_nor_t`、以降足回り固有の引数を(存在する場合)渡す。

## クローラー

クローラーとは日本語で無限軌道、履帯、キャタピラー、カタピラなどと呼ばれ、モーターの出力を受ける起動輪、張り具合を調節する遊動輪、接地する転輪などを囲うように板を繋げて一つの輪にしたものである。
本ライブラリにおいては左右に一つずつある場合をサポートしている。

操作方法は2つ用意されており、`crawler_ctrl_type_t`列挙型に定義されている。
`CRAWLER_LEFT_STICK`は左スティックのみで操作する方法で、上下が前後移動、左右が左右回転に割り当てられている。
`CRAWLER_BOTH_STICK`は左右スティックで操作する方法で、左スティック上下が左クローラーの前後、右スティック上下が右クローラーの前後にそのまま割り当てられている。

速度計算用関数は`calc_speed_crawler`関数で、固有の引数として`crawler_ctrl_type_t`を受け取る。

## メカナムホイール

メカナムホイールとは、某掃除屋のドーナツチェーンの人気商品に少し似ている見た目のタイヤである。~どう見てもフレンチクルーラーである。~タイヤ全体に対し45deg傾いて配置されているローラーがミソであり、これのおかげで通常のタイヤでは不可能である左右方向への並進移動を可能としている。
四輪のタイヤをロボット中心から同じ距離となるよう配置することでうまく動作する。正方形に配置する他、長方形に配置しても概ねよく動作する。
オムニホイールに比べ、モーターの回転方向と前後方向が一致するため、前後に全速力で移動する場合などにおいてはより優れていると言える。

速度計算用関数は`calc_speed_mechanum`(※)関数で、固有の引数はない。

(※)
関数名には`mechanum`と入っているがこれは誤りであり、正しくは`mecanum`である。これは将来のリリースにおいて修正される可能性があることに注意されたし。

## オムニホイール

オムニホイールとは、~ポン・デ・リングである。~某掃除屋のドーナツチェーンの人気商品に少し似ている見た目のタイヤである。タイヤ全体に対し直交する方向に配置されているローラーがミソであり、これのおかげでタイヤに直交する方向への移動が可能となっている。
他のタイヤとは異なり、全てのタイヤがロボット中心から見た時同じ向きとなるよう配置する。つまり、最終的に円形となるように配置される。
本ライブラリにおいては任意の輪数(無論、3以上)をサポートしている。

速度計算用関数は`calc_speed_omni`関数で、固有の引数として輪数と一輪目の配置角度(※)を受け取る。

(※)
角度は右を0とし、反時計回りに増加するものとする。また、「一輪目」の定義について、恐らくどこでも良いと思われるが、右から反時計回りに見ていき最初に現れたタイヤとすることを想定している。
